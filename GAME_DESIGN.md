# 蛋宝世界（DANBO World）游戏设计与现有内容说明

> 更新时间：2026-06-16
> 当前版本：v20260616.7
> 运行方式：静态 Web / Three.js / GitHub Pages

## 1. 游戏定位

蛋宝世界是一个低门槛、可爱风格的 3D 网页小游戏集合。核心目标是：

- 让玩家用简单圆润的“蛋宝/动物蛋”角色探索多个主题城市。
- 在城市里收集金币、和 NPC 互动、抓取/投掷/打斗、进入赛道和隐藏区域。
- 保持画面轻量、明亮、柔和、偏卡比式可爱；角色与建筑不追求写实细节，而追求圆润、干净、易读。
- 手机与桌面都能直接在浏览器游玩，无需安装 App。

## 2. 技术栈与部署

| 模块 | 当前实现 |
|---|---|
| 3D 引擎 | Three.js r160，通过 CDN 动态加载，失败时使用备用地址 |
| WASM 试点 | `wasm/danbo_core.wasm`，目前用于传送距离/高度/近地面/入口状态、通用命中/AABB/音效衰减等纯判定；失败自动回退 JS |
| 页面形态 | 纯静态 HTML/CSS/JS，无构建步骤 |
| 渲染风格 | Toon / Pastel / 软萌低多边形，动态阴影、雾、太阳光、少量后处理 |
| 音频 | Web Audio API 程序化 BGM 与音效，无外部音频文件 |
| 多语言 | 简体中文、繁体中文、日语、英语；根据 localStorage 或浏览器语言决定 |
| 移动端 | 虚拟摇杆与按钮；默认在菜单/选人时隐藏，进入游戏才显示 |
| 部署 | GitHub Pages，自定义域名 `eggy.ff18.com` |
| 版本规则 | `vYYYYMMDD.N`，静态资源用查询参数强制刷新缓存 |

## 4. 游戏流程

1. **加载画面**：最开始显示本地化的“3D 引擎加载中 / Loading 3D engine”。
2. **开始画面**：显示标题、版本号、开始按钮。
3. **选人画面**：SF2 风格角色选择，高清 Canvas 头像。
4. **城市探索**：玩家出生在当前主题城市，可自由移动、跳跃、战斗、收集金币。
5. **传送/入口**：靠近水管、传送门、巴别塔、云中界管道时出现确认框。
6. **竞速/平台关/月面战场**：通过入口进入不同玩法，结束后返回城市。

## 5. 角色系统

当前共有 8 个可选角色。角色视觉统一为简单圆润的软萌蛋形/动物蛋，不做复杂写实脸部细节。

| 角色 | 类型 | 原型定位 | 代表能力/特殊技 |
|---|---|---|---|
| 经典蛋宝 | egg | Ryu | 波动拳、升龙拳、旋风腿 |
| 野牛 | bull | E.Honda | 百裂掌、头槌冲撞 |
| 猫仔 | cat | Blanka | 电击、滚动攻击 |
| 鸡公 | rooster | Guile | 音速手刀、闪光飞踢 |
| 狗仔 | dog | Ken | 波动拳、火焰升龙拳、旋风腿 |
| 马骝 | monkey | Chun-Li | 气功拳、百裂脚、回旋鸟踢 |
| 大熊 | bear | Zangief | 双回旋臂、螺旋打桩 |
| 曱甴 | cockroach | Dhalsim | 瑜伽火球、瑜伽火焰、长手长脚 |

### 角色视觉原则

- 身体以球体/蛋体为主，五官大、干净、少细节。
- 每个角色只保留最容易识别的动物特征：耳朵、角、冠、触角等。
- 避免复杂肌肉、皱纹、过多脸部线条，保证“可爱”和“清晰”。

- 左侧虚拟摇杆移动。
- 右侧按钮：抓、跳、拳、脚、聊天、视角、视角重置。
- 菜单/选人画面默认不显示虚拟按钮，进入城市/赛跑/平台关后自动显示。

## 7. 相机系统

当前城市探索支持 3 种视角：

1. **普通俯视跟随**：默认视角，适合探索和收集。
2. **第三人称跟随**：类似越肩视角，可鼠标/触摸拖拽调整，带建筑遮挡处理。
3. **第一人称**：隐藏本地玩家 Mesh，将相机放到角色眼睛附近，适合网页端沉浸查看世界。

月面还保留旁观者自由相机，用于观察大规模 MS 战场。

## 9. 传送系统

### 已有传送方式

| 类型 | 说明 |
|---|---|
| 竞速传送门 | 城市内 12 个入口，对应 12 条赛道 |
| 横版平台关入口 | 城市内蘑菇入口，进入蛋宝冒险横版关 |
| 透明水管 | 城市之间传送，带飞行动画 |
| 巴别塔 | 金币数量达到条件后出现，可上下往返云中界 |
| 云中界月球管道 | 从云中界进入月面都市 |
| 月面返回传送门 | 从月面返回之前的地面城市 |
| 聊天指令 | `/fly me to the moon`、`/sakura`、`/snow` |

### 传送交互规则

- 靠近入口显示提示，继续靠近弹出确认对话框。
- 现在传送触发不再严格依赖 `onGround`，避免在斜面、管道边缘、小台阶或轻微弹跳后“走进没反应”。
- 传送距离已适当放宽，水管与传送门更容易触发；非水管入口使用更大的视觉确认圈，避免“看到提示但没有弹窗”。
- 已审计入口：12 个竞速传送门、横版平台关入口、城市透明水管、巴别塔上下电梯、云中界月球管道、月面返回地球传送门、聊天传送指令。

## 10. 竞速系统

当前有 12 个竞速入口/赛道主题：

1. 疯狂赛道：旋转臂、传送带。
2. 锤子风暴：大锤、摆锤。
3. 极限挑战：高速度障碍。
4. 冠军之路：最终竞速。
5. 绿宝石山丘：Sonic 风格金币与弹簧。
6. 火焰山谷：加速带与熔岩。
7. 冰霜滑道：冰面与弹簧。
8. 彩虹天空：空中平台与金币雨。
9. 蘑菇王国：经典水管与板栗敌人。
10. 岩浆城堡：岩浆与火球。
11. 云端天堂：空中平台与弹簧。
12. 库巴城堡：最终综合障碍。

竞速包含 NPC 对手、排名、金币、终点、回城流程和 Bifrost 风格入场动画。

## 11. 平台关系统

平台关是横版玩法，复用蛋宝角色与基础物理，包含：

- 地块、缺口、砖块平台、问号块。
- 蘑菇、星星、火球等道具/效果。
- Goomba/Koopa 风格敌人。
- 移动平台、坍塌平台、旋转平台、风区、岩浆。
- 侧视相机和独立 HUD。

## 12. 战斗与互动

### 基础互动

- NPC 可被抓取、投掷、打飞、击晕。
- 玩家可抓物件、投物件、蓄力投掷。
- 空中下压/抱摔/打桩等会造成击飞与眩晕。
- 城市场景中的部分物体可被碰撞、抛出或作为障碍。

### 战斗系统

- 轻/中/重打击、投掷、特殊技都会累积眩晕值。
- 眩晕后出现挣扎提示，可通过方向键/摇杆挣扎。
- 各角色有不同特殊招式触发方式，如前前+拳、后前+脚、R+T、前后前+F 等。

## 13. 收集与事件

- 城市内生成大量金币；喷泉周围有额外金币。
- NPC 可以与金币、玩家、物体产生互动。
- 金币达到条件会触发巴别塔事件。
- 有 SOTN 风格区域名揭示，用于云中界和城市切换提示。

## 14. 月面都市与 MS 战场

月面是当前最复杂场景之一：

- 包含冯·布劳恩与格拉纳达两个城市区域。
- 城市内有 AT 力场半球护盾，带入口门和保护规则。
- 战场区有大量机体：Gundam/GM/Zaku/Dom 等，以及大型舰船。
- 武器包含光束、导弹、爆炸等视觉效果。
- 城市区内削弱/隐藏危险战斗效果，避免干扰普通探索。
- 支持旁观相机观察大规模战斗。

## 15. 视觉方向

当前视觉目标：**可爱、柔和、粉彩、低复杂度、高可读性**。

- 角色：简单圆润，接近卡比式亲和力。
- 建筑：保持简单几何体，不做写实 3A 建筑细节。
- 光照：柔和日光、较高环境光，避免强对比让角色变脏。
- 色彩：以糖果色/粉彩为主，避免过重阴影。
- 性能：减少不必要材质复杂度，保留物体数量，优先用实例/低面数/简材质优化。

## 16. 音频系统

- 所有 BGM 和音效均由 Web Audio API 实时生成。
- 支持音乐与音效开关。
- 移动端有音频解锁逻辑，避免浏览器自动播放限制。
- 不依赖外部 mp3/ogg 文件。

## 17. 国际化

当前支持：

- 简体中文：`zhs`
- 繁体中文：`zht`
- 日语：`ja`
- 英语：`en`

加载画面、开始按钮、选人、HUD、传送确认、聊天占位、按钮文案等都尽量使用当前语言。

## 19. 后续建议

1. **继续统一可爱美术风格**：减少角色脸部奇怪细节，强化大眼、腮红、圆润轮廓。
2. **分层优化性能**：不减少物体数量的前提下，用实例化、材质复用、LOD、批处理优化。
3. **整理旧 `game.js` 与 `js/` 拆分逻辑**：确认是否还需要保留旧整合文件，避免维护混乱。
4. **补自动测试脚本**：至少覆盖加载、选人、进城、视角切换、水管传送、竞速入口。
5. **继续补文案 i18n**：HUD 新按钮、特殊提示、错误提示全部进入统一词典。
